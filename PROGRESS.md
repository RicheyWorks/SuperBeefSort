- **WikiSort comparison-constant cut (threaded distinctness)** ‚Äî the all-distinct gate is no longer recomputed from scratch at every merge. `WikiSortStrategy` threads a monotone `trustDistinct` flag (seeded by a one-time internal-duplicate scan of the insertion-sorted base runs): while it holds, every run produced so far is strictly increasing, so a merge decides distinctness with a single *cross-run-equal* walk (`crossRunEqual`, one comparison per element) instead of the two-comparison `fullyDistinct` (which also re-verifies within-run order). The first duplicate seen anywhere clears the flag, after which the exact, self-contained `fullyDistinct` gate is used ‚Äî so the block-vs-rotation choice (hence the output, the move count, and stability) is provably identical to the old per-merge gate on every input; only the comparison count falls. - **Memory-budget filter for SMART (`CostModelStrategySelector`)** ‚Äî the `AuxMemory` capability now shapes the `SMART` objective too, not just `STABLE`. `CostModelStrategySelector` gained an optional `smartAuxBudgetBytes` cap (new constructor; default `Long.MAX_VALUE` = unlimited, so the no-arg selector is byte-for-byte the old behaviour). - **Duplicate-tolerant `merge.wiki` (faithful WikiSort port)** ‚Äî `WikiSortStrategy` is now a faithful, no-cache port of BonzaiThePenguin's WikiSort (the Kim&nbsp;&amp;&nbsp;Kutzner block merge), replacing the previous distinct-only fast path + rotation fallback. - **Measured peak auxiliary memory** ‚Äî strategies now self-report their scratch allocation through a new `SortBuffer.recordAux(bytes)` meter (peak-tracked, mirroring the comparison/move metering), surfaced as `SortResult.peakAuxBytes` and shown in `SortReport`.
