# Phase 4a — offline-trained strategy selection

This directory holds the **offline training + model-export** half of Phase 4
(`docs/adr-phase4-python-intelligence.md`, action item 3): train a compact
classifier that predicts the cheapest sort strategy from a `DataProfile`, and
export it in a stable, versioned format a future in-process
`LearnedModelStrategySelector` (action item 4) loads — no runtime service, no
network on the hot path.

## Why this can beat the heuristics

The gate (action item 2) measured real gaps against a brute-force oracle on 324
workloads:

| selector | exact-match | mean regret |
|---|---|---|
| `CostModelStrategySelector` | 60.5% | 386.52% |
| `BanditStrategySelector` | 65.4% | 191.94% |
| **learned tree (this dir)** | **98.1%** | **0.50%** |

(Learned numbers are on the **same held-out 324 gate workloads**, predictions
masked to feasible strategies; corpus generated by the validated mirror.) The
six residual misses are all one shape — `sawtooth` comparable-only, oracle
`jdk.timsort` vs predicted `intro` — with bounded regret (22–32%), exactly the
kind of case the selector's confidence gate defers to the cost-model delegate on.

## Files

| file | role |
|---|---|
| `sbs_mirror.py` | Faithful Python mirror of `java.util.Random`, the profiler, all 8 candidate strategies' **metering**, and the oracle. Validated to reproduce the gate's exact spread. |
| `validate_gate.py` | Reproduces the gate's 324-workload oracle spread (`counting 138, jdk.timsort 108, insertion 48, intro 30`) and asserts the match — the fidelity gate for the mirror. |
| `gen_corpus.py` | Emits `phase4_train.csv` (disjoint-by-size grid) + `phase4_gate_test.csv` (the 324 gate workloads), schema-identical to the Java dumper. |
| `train_selector.py` | Trains the classifier, scores it on the held-out gate set vs the published baselines, and exports the model. Verifies both walkers round-trip sklearn's predictions (324/324). |
| `sbs_selector_model.json` | The exported model (schema v1), human-readable. |
| `sbs_selector_model.txt` | The **same** model in a flat, line-oriented format — this is what the Java `SelectorModel` loads (the engine ships no JSON parser). A copy lives in `src/test/resources/` for the selector test. |

## Two corpus sources (same schema)

- **Python mirror (in-sandbox, used here):** `python3 gen_corpus.py`. Validated
  against the gate, so labels are real; `distinct_estimate` stands in the exact
  distinct count for the HLL estimate.
- **Java dumper (host, production parity — recommended for the shipped model):**
  `./gradlew phase4Corpus` writes `build/reports/phase4-corpus-{train,gate}.csv`
  using the **real** profiler features (HLL distinct, sampled inversions,
  distribution class). Train on those for serving-time feature parity:

  ```bash
  cd tools/phase4
  python3 train_selector.py \
      ../../build/reports/phase4-corpus-train.csv \
      ../../build/reports/phase4-corpus-gate.csv
  ```

## Reproduce (in-sandbox)

```bash
cd tools/phase4
python3 validate_gate.py     # assert the mirror matches the gate spread
python3 gen_corpus.py        # build phase4_train.csv + phase4_gate_test.csv  (~50s)
python3 train_selector.py    # CV, gate-benchmark score, export sbs_selector_model.json
```

## Model export schema (v1) — the SBS ↔ Java contract

`sbs_selector_model.json` is a flat decision tree; the Java loader walks it with
no ML dependency:

```jsonc
{
  "schema_version": 1,
  "model_type": "decision_tree",
  "feature_columns": ["size", "sortedness_ratio", ...],  // order IS the contract
  "classes": ["counting", "insertion", "intro", "jdk.timsort"],
  "n_nodes": 23,
  "nodes": {
    "feature":    [int, ...],   // index into feature_columns; -1 at a leaf
    "threshold":  [double, ...],// go LEFT when feature <= threshold
    "left":       [int, ...],   // child node id; -1 at a leaf
    "right":      [int, ...],
    "class":      [int, ...],   // argmax class index (read at leaves)
    "confidence": [double, ...] // leaf purity, for the selector's confidence gate
  }
}
```

### Java loader + selector (action item 4 — built)

The flat file (`sbs_selector_model.txt`) is the same tree as line-oriented records
(`key value...`), so the Java loader needs no JSON dependency:

```
sbs-selector-model 1 decision_tree
features size,sortedness_ratio,...
classes counting,insertion,intro,jdk.timsort
n_nodes 23
feature 6 11 0 -1 ...
threshold 1.5 119.0 9000.0 0.0 ...
left 1 2 3 -1 ...
right 18 9 4 -1 ...
class 0 3 1 1 ...
confidence 0.40 0.58 0.89 1.0 ...
```

Shipped in `select/`:

- **`SelectorModel`** — `load(Reader|Path)` / `parse(String)` / `fromClasspath(...)`
  (Optional). `predict(double[])` walks the tree to a leaf:
  ```java
  int node = 0;
  while (feature[node] >= 0)
      node = x[feature[node]] <= threshold[node] ? left[node] : right[node];
  return new Prediction(classes.get(clazz[node]), confidence[node]);
  ```
- **`LearnedModelStrategySelector`** — a `LearningStrategySelector` that wraps a
  `CostModelStrategySelector` delegate and overrides its pick only when
  (1) policy is `SMART`, (2) `size >= sizeGate` (default 256), (3) leaf
  `confidence >= confidenceMargin` (default 0.65), and (4) the predicted strategy
  is registered **and applicable** (integer-key + counting-feasibility gates) —
  else it returns the delegate's plan verbatim. Its `DataProfile -> double[]`
  mapping is the Java twin of `gen_corpus.features`. Use it like any selector:

  ```java
  var model = SelectorModel.load(Path.of("build/reports/.../sbs_selector_model.txt"));
  var selector = new LearnedModelStrategySelector(model); // default delegate + gates
  BeefSort.with(cmp).source(data).keyEncoder(enc).policy(SMART).selector(selector)...;
  ```

Covered by `LearnedModelStrategySelectorTest`. Open: item 5 — benchmark it in-JVM
(JMH) and confirm the inference cost is amortized before promoting it as a default.
